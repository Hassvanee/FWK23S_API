Dokumentation av det här APIet

Routes

  • GET /users - returnerar alla användare.

  • GET /users/{id} - returnerar en user med angivet id eller status 404 om användaren saknas.

  • POST /users - skapar en ny användare accepterar JSON objekt på formatet {"username": "unikt namn", "firstname": "", "lastname": "", "password": ""}. username är obligatoriskt och ska vara unikt.

  • POST /login - för inloggning. Returnerar en JWT som används som bearer token i anrop till routes skyddade med auth. Accepterar JSON-objekt på formatet {"username": "", "password": ""}.

  • PUT /users/:id - uppdaterar användaren med angivet id, accepterar JSON-objekt på formatet {"firstname": "", "lastname": "", "password": ""}.
